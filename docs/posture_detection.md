# 坐姿矫正 · 设计文档

## 1. 目标与范围

实时、连续、**本地免费**地监测小朋友的坐姿，发现问题温柔提醒（建立信心，
不打击）。本阶段只做坐姿；握笔检测复用同一套「关键点 → 特征 → 判定」框架，后续接入。

判定三件事（够覆盖 80% 的常见毛病）：
1. **离得太近 / 趴太低**（凑近桌面、护眼）
2. **低头**（头垂向桌面）
3. **身体歪斜**（单边肩膀塌、整个人歪）

## 2. 两层结构

```
现成模型给前半段：摄像头 → MediaPipe Pose → 33 个关键点
你写后半段：     关键点 → 几何特征 → 门限判定 → 节流 → 提醒
```

没有「这姿势对不对」的通用现成模型，因为「标准坐姿」依赖机位和身材比例。
所以后半段是**基于关键点的几何规则**，而不是一个可下载的网络。
这恰好是信号处理的主场：**特征提取 + 门限判决**（像 VAD / 能量检测）。

## 3. 用到的关键点（MediaPipe Pose 33 点的子集）

| 名称 | 索引 | 用途 |
|------|------|------|
| nose | 0 | 头的竖直位置 |
| left_eye / right_eye | 2 / 5 | （备用，握姿/距离细化） |
| left_ear / right_ear | 7 / 8 | （备用，头前倾细化） |
| left_shoulder / right_shoulder | 11 / 12 | 归一化基准、肩膀倾斜 |

## 4. 三个特征（全部尺度无关，除了刻意保留的那个）

设归一化图像坐标，x 向右、y **向下**，均在 [0,1]。

- **shoulder_width** = ‖左肩 − 右肩‖
  - 这是唯一**保留绝对尺度**的量。人离摄像头越近，它越大——所以它本身就是「凑近」信号。
- **neck_vertical** = (肩中点_y − 鼻_y) / shoulder_width
  - 端坐时鼻子在肩上方，为正值；**低头时鼻子下移，该比值变小**。
- **shoulder_tilt** = (右肩_y − 左肩_y) / shoulder_width
  - 带符号；偏离基准越大说明身体越歪。

> 用 shoulder_width 归一化的好处：小朋友单纯坐远/坐近时，neck_vertical 和
> shoulder_tilt 几乎不变，避免误报。

## 5. 个性化标定（核心设计：相对，不绝对）

**不要写死绝对角度。** 先采「这孩子坐端正时」的基准，之后所有判断都是**相对偏移**。

- 做法：让她坐好，采 ~30 帧特征，取平均得到 `Baseline(shoulder_width, neck_vertical, shoulder_tilt)`，存 JSON。
- 换机位 / 换小孩 → 重新标定 10 秒，零改代码。
- 亲子参与点：「来，坐端正，让台灯记住你最棒的样子。」

类比：先测噪声基底 / 做通道校准，再检测偏离。相对量永远比绝对量稳。

## 6. 判定逻辑（`posture_rules.judge`）

| 问题 | 触发条件 | 默认阈值 |
|------|----------|----------|
| `too_close` | shoulder_width / 基准 > `proximity_ratio` | 1.20（近 20%） |
| `head_down` | neck_vertical < 基准 × `head_drop_ratio` | 0.75 |
| `leaning` | \|shoulder_tilt − 基准\| > `shoulder_tilt_delta` | 0.12 |

每个问题带一个 `severity`（超过阈值多少，0~1），可用于决定提醒的轻重。
阈值都在 `config/posture.yaml`，调参不动代码。

## 7. 防唠叨节流（`monitor.PostureMonitor`）

逐帧判定会抖动、会唠叨。节流器把「检测」变成「该不该提醒」：

- **持续**：问题要连续保持 `persistence_s`（默认 3s）才第一次提醒——滤掉一闪而过。
- **冷却**：同一类问题两次提醒至少间隔 `cooldown_s`（默认 30s）——不烦人。
- 时钟通过参数注入（`now`），所以可用假时钟做单测，无需真等待。

这就是产品设计里「多久提醒一次才不烦人」的工程化。

  但几何规则已解决大部分且**可解释**，不急。贴标签又是个亲子活动。
- **握笔检测**：换成 MediaPipe Hands，沿用本框架（特征→门限→节流）。
- **CJK 上屏**：OpenCV `putText` 不支持中文，demo 暂在画面上用 ASCII 状态、
  中文提醒打到控制台。正式 UI 用 PIL/Qt 绘制中文。
