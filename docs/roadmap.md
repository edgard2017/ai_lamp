# 路线图（分阶段，每阶段一个可演示成果）

节奏：一次只做一个小可验证产出，不堆任务。别让机械/外壳挡住软件。

## Stage 0 · 环境就绪
- Pi 刷 64 位系统，接屏幕/摄像头/麦/喇叭。
- 跑通：摄像头画面上屏 + 录一句话能放出来。
- 👧 小朋友：对着摄像头做鬼脸、录自己的声音。

## Stage 1 · 坐姿矫正  ←（当前）
- MediaPipe Pose → 关键点 → 三特征（低头/凑近/歪斜）→ 标定 → 节流提醒。
- ✅ 产出：弯腰驼背时，台灯说「坐直一点点会更舒服哦」。
- 👧 小朋友：故意驼背测试灵不灵，帮你「找 bug」；标定时「坐端正让台灯记住」。
- 代码：`src/ai_lamp/vision/`，demo：`apps/posture_demo.py`。

## Stage 2 · 握笔检测
- MediaPipe Hands（21 点）→ 复用「特征→门限→节流」框架。
- 先从「检测到手在写字区」做起，握姿规则逐步加。

## Stage 3 · 语音对话
- STT：faster-whisper（本地小模型）或云端。
- TTS：piper（本地、轻、音色自然）。
- ✅ 产出：小朋友说「你好」，台灯回应。

## Stage 4 · 作业引导（项目灵魂）
- 灯头拍题 → VLM（先豆包验交互，后可自建 Qwen-VL）识别题目。
- **苏格拉底式引导，不直接给答案**；答对夸奖建立信心，卡住拆成更小提示。
- ✅ 产出：陪小朋友做完一道题的完整对话。

## Stage 5 · 错题集 + 能力画像
- 引入本地 SQLite 数据层：`sessions / questions / attempts`。
- 每道错题记**错误类型**（概念不懂/计算粗心/读题错/不会下手）> 只记对错。
  错误类型在引导对话里顺手就采到了。
- LLM 按需：录入打标签 + 每周生成「能力周报」给家长。
- 延伸：间隔重复复习、画像回灌到引导。
- **关键**：Stage 4 一开工就建表开始记录，别等做分析了才补日志。
